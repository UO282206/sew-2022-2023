Cambios

1.0

Primera versión estable de Ruby.

1.8

Tras el lanzamiento de Ruby 1.8, el Centro de Promoción de Estándares Abiertos de la Agencia de Promoción de las Tecnologías de la Información (una agencia gubernamental japonesa) desarrolló las especificaciones de lenguaje de Ruby y las entregó al Comité Japonés de Estándares Industriales y a la Organización Internacional para la Estandarización (ISO). Fue aceptado como estándar industrial japonés y como estándar internacinal.

1.9

Ruby 1.9 introdujo las variables locales de bloque, una sintaxis lambda adicional, una sintaxis literal hash adicional, soporte para la codificación de caracteres por string, una nueva API de socket (con soporte para IPv6) y una herramienta de importación.

2.0

Se añadieron los argumentos de método keyword, un nuevo método para ecternder una clase, un nuevo literal para la creación de un array de símbolos, una nueva API para la evaluación perezosa de enumerables y una nueva convención para convertir objetos en hashes.

2.1

La política de versionado de Ruby pasó a ser parecida al versionado semántico, aunque no igual, pues es incompatible con este.

2.2

Se realizaron mejoras para aumentar la eficiencia del lenguaje, arreglos de bugs y actualizaciones de librerías, así como la eliminación de APIS deprecadas. A su vez, Ruby 2.2 introdujo cambios significativos en el tratamiento de la memoria tales como un garbage collector incremental, garbage collection de símbolos y nuevas opciones de jemalloc.

2.3

Ruby 2.3 introduce la habilidad de marcas todos los string literals como "congelados" por defecto para mejorar la eficiencia de las operaciones con strings, la comparación de hash y un operador de navegaciñon segura.

2.4

Binding, Unicode case mappings y detección de deadlocks en los hilos.

2.5

Statements de rescue y ensure, method chaining, soporte para cobertura de branch y transformaciones hash más fáciles.

2.6

JIT experimental y máquina virtual.

2.7

Pattern matching, mejoras de REPL y separación de argumentos keyword y preposicionales.

3.0

JIT.

3.1

YJIT (nuevo compilador de JIT desarrollado por Shopify), sintaxis hash abreviada, block forwarding anónimo, mejoras en los pin operators y un nuevo debugger.
