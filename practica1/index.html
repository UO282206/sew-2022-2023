Preguntas generales

¿Qué es Ruby?

¿De dónde procede el nombre "Ruby"?

Influenciado por otro lenguaje de programación, Perl, Matz quería que su nuevo lenguaje recibiera su nombre de una gema. Escogió el rubí por ser la piedra de nacimiento de uno de sus colegas.

Más adelante, Matz se dio cuenta de que el rubí viene después de la perla en una serie de situaciones. En las piedras de nacimiento, la perla representa el mes de junio mientras que el rubí representa julio. En tamaños de fuente, perla es 5pt y rubí es 5.5pt. Él pensó, por tanto, que Ruby sería un buen nombre para un lenguaje más nuevo y ostensiblemente mejor que Perl.

¿Cuál es la historia de Ruby?

Ruby nació el 24 de febrero del año de 1993, cuando Matz tuvo una conversación con un colega sobre la posibilidad de crear un lenguaje de scripting basado en la programación orientada a objetos.

Matz conocía Perl, pero no le gustaba porque le daba la impresión de ser un "toy language". También conocía Python, pero le disgustaba que las características de orientación a objetos parecían un añadido. Al final, se decidió a crear su propio lenguaje.

Le tomó meses conseguir que el intérprete se ejecutara y le añadió a su nueva creación ciertas características que deseaba incluir tales como iteradores, tratamiento de excepciones y "recolección de basura", pero finalmente la versión 0.95 de Ruby sería presentada ante grupos de noticias japoneses en diciembre de 1995.

Instalando Ruby

¿Qué sistemas operativos soportan Ruby?

Dado que Ruby fue desarrollado bajo Linux y está escrito en un C bastante puro, Ruby puede funcionar en sistemas Linux y otros sistemas estilo UNIX tales como Windows o macOS.

¿Dónde puedo obtener Ruby?

La última versión de Ruby puede ser descargada desde https://www.ruby-lang.org/en/downloads/. La página también contiene links a otros sitios de descarga.

¿Puedo hacerme con el árbol fuente de desarrollo?

Si tiene un cliente CVS, puede ver el actual árbol fuente de desarrollo usando:


                $ cvs -d :pserver:anonymous@cvs.netlab.co.jp:/home/cvs login
                (Logging in to anonymous@cvs.netlab.co.jp)
                CVS password: guest
                $ cvs -d :pserver:anonymous@cvs.netlab.co.jp:/home/cvs co ruby

Si no posee un cliente, CVS puede obtener una versión actualizada del desarrollo a partir del siguiente enlace: https://cache.ruby-lang.org/pub/ruby/snapshot.tar.gz.

¿Cómo compilo Ruby?

Bajo Unix, Ruby usa el sistema autoconf para configurar el entorno de la build. No necesitas el comando autoconf para hacer la build de Ruby desde una distribución. Solo necesitas usar los comandos: 


                $ ./configure [configure options]
                $ make
                $ make test
                $ make install

Podrías necesitar privilegios de superusuario para instalar Ruby si no haces override a la localización de instalación por defecto (/usr/local). Puedes obtener la lista completa de las opciones de configuración usando: 


                $ ./configure --help

Si trabajas desde el repositorio fuente, puedes necesitar ejecutar autoconf antes de ejecutar configure.

¿Cómo le digo a Ruby dónde están mis librerías?

En algunos sistemas, el proceso de build puede fallar a la hora de encontrar las librerías usadas por los módulos de extensión (por ejemplo, las librerías dbm).

Puedes decirle a Ruby que encuentre librerías con: 

                $ ./configure --with-xxx-yyy=DIR

donde XXX puede ser


                opt           extra software path in general
                dbm           path for dbm library
                gdbm          path for gdbm library
                x11           ...for X11..
                tk            ...for Tk...
                tcl           ...for Tcl...

e YYY puede ser


                dir           specifies -I DIR/include -L DIR/lib
                include       specifies -I DIR
                lib           specifies -L DIR

En HP-UX puede haber problemas haciendo la build con gcc. Intenta usar el compilador nativo en su lugar. Tetsuya Watanabe recomienda: 

                $ CC="cc -Ae" CFLAGS=-O ./configure --prefix=/opt/gnu

También puede haber problemas con el sed nativo de HP. Es recomendable instalar el equivalente de GNU.

¿Hay binaries precompiladas disponibles?

Hay disponible en RubyInstaller una descarga única que contiene todo lo que necesitas para ejecutar Ruby en varios sistemas operativos Windows.
