Cómo se compara Ruby con...

Python

  • Similaridades
    1. Los dos son lenguajes de alto nivel.
    2. Los dos son lenguajes de scripting del lado del servidor.
    3. Ambos se usan para el desarrollo de aplicaciones web.
    4. Ambos son lenguajes multiplataforma.
    5. Ambos tienen una sintaxis limpia y fácil de leer.
    6. Ambos utilizan un prompt interactivo llamado IRB.
    7. Los objetos son de tipado strong y dinámico en ambos lenguajes.
    8. Ambos utilizan herramientas de documentación embebida.
    9. Hay disponible para cada lenguaje un estilo de debugger de GNU.
    10. Ambos lenguajes tienen soporte para modos Emacs.
  • Diferencias
    1. Ruby es un lenguaje de programación totalmente orientado a objetos.
    2. Solo Eclipse da soporte para desarrollar en Ruby.
    3. Se pueden usar mixins en Ruby.
    4. Ruby tiene un conjunto de librerías más pequeño que Python.
    5. En Ruby, una vez que le asignas un valor a una variable, no se le puede desasignar.
    6. Ruby permite el uso de bloques y procs.
    7. Ruby permite hacer funciones lambda más extensas.
    8. En Ruby no hay funciones.
    9. Ruby es un lenguaje centrado en el desarrollo web.
    10. En Ruby se permite el uso de statements del tipo switch/case.
    11. Ruby permite modificar las clases que ya vienen implementadas.
    12. Ruby solo permite herencia única.
    13. Ruby no permite el uso de tuplas.

Perl

  • Similaridades
    1. En Ruby tienes un sistema de manejo de paquetes similar a CPAN llamado RubyGems.
    2. Ambos lenguajes tienen soporte para expresiones regulares built-in.
    3. Muchas de las funcionalidades más usadas son built-in.
    4. Los paréntesis suelen ser opcionales.
    5. Los strings funcionan casi de la misma manera en ambos lenguajes.
    6. La expresión del comando de la shell usa `backticks`.
    7. Ambos tienen herramientas de documentación embebida.
  • Diferencias
    1. Ruby es un lenguaje de programación totalmente orientado a objetos.
    2. El número de librerías presente en Ruby es mayor que el presente en Perl.
    3. El soporte de Unicode en Ruby no es tan fuerte como en Perl.
    4. El soporte de las expresiones regulares es menor en Ruby que en Perl.
    5. Ruby solom tiene un tipo de variable.
    6. El desempeño de Ruby en ejecución es peor que el de Perl.
    7. El punto fuerte de Ruby es la metaprogramación.
